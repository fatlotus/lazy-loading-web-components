What is going on?

The two WebComponents below were loaded dynamically.

What is this trickery?

Right now, the community seems to be coalescing around the use of Bower to statically import and "vulcanize" the custom elements being used in a page. To get a Material Design button, for example, one must run:

$ bower install --save PolymerElements/paper-button`

and then import:

<link rel="import" href="bower_components/paper-button/paper-button.html">

for each component you plan to use on a page. Using dynamic loading, however, there is just a single import to get the entire suite.

<link rel="import" href="my-library.html">

Why does this matter?

It's easier to use, but more importantly, it allows for the creation of much larger catalogs of elements. Imagine an NPM, but for HTML elements &emdash; where you can customize code, and then copy and paste into your page, without downloads, imports, or tweaks.
